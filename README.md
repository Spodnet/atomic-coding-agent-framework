# Atomic Coding Agent Framework

This setup transforms an AI agent from a standard code generator into a disciplined **Recursive Architect**. By enforcing Atomic TDD (Test-Driven Development) and Domain-Driven Design, it ensures that every line of code generated by the agent is verified, secure, and part of a pre-approved architectural plan.

## Overview

The core philosophy is based on **Atomic TDD & Recursive Decomposition**. Instead of writing high-level logic in a single pass, the agent decomposes every feature into "Functional Atoms" (small units of <20 lines of code).

### The Atomic Execution Loop

For every leaf-node task, the agent is instructed to follow this exact sequence:

1. **Interface & Purpose**: Define the function signature and a single-sentence description.
2. **File Path & Rationale**: State the destination and why it belongs there.
3. **Test Creation**: Write a specific unit test (including happy paths, adversarial cases, and security checks).
4. **Proof of Failure (Red)**: Run the test to prove it fails without implementation.
5. **Minimal Implementation**: Write only the code necessary to make the test pass.
6. **Verification (Green)**: Run the test again to prove it works.
7. **Atomic Commit**: Commit the changes immediately.

## How to Use with Coding Agents

This framework provides specialized workflows that you can trigger using commands when interacting with the coding agent.

### Available Commands

- **/atomic**: Trigger the atomic workflow. Use this when you have a new feature request. The agent will analyze your request and break it down into the smallest possible testable units, presenting a Task List for your approval before writing any implementation code.

**Example Prompt:**

```text
/atomic I need to build a new authentication system with login, logout, and password reset functionalities.
```

- **/atomic-step**: Process an individual step. Use this to execute a single atomic task from the approved Task List.

**Example Prompt:**

```text
/atomic-step [step-number]
```

e.g.

```text
/atomic-step 1
```

- **/atomic-auto**: Process all atoms automatically. Once a Task List is approved, use this command to authorize the agent to execute the implementation plan sequentially. The agent will follow the complete Atomic TDD sequence (Test -> Fail -> Code -> Pass -> Commit) for each task autonomously, stopping only if an error cannot be resolved.

**Example Prompt:**

```text
/atomic-auto
```

- **/atomic-registry-sync**: Sync the registry for any atoms that already exist in the codebase. Useful to rebuild the document if it gets out of sync with the codebase.

**Example Prompt:**

```text
/atomic-registry-sync
```

## Constraints & Organization

- **Atomic Commits**: The agent is restricted to one atom per commit.
- **No Hardcoded Paths**: Environment variables and paths must use dependency injection.
- **Strict Typing**: Interfaces must adhere to full type hints.
- **Regression Testing**: The full test suite runs after every atomic execution.
- **Documentation**: Branches aren't complete until relevant `.md` documentation files (in the `docs/` folder) are updated.
